超大日志处理工具（HTML 版）完整描述

一、核心功能
该工具是一款面向 OHOS 格式日志的 Web 端处理分析工具，核心解决超大日志文件（GB 级）的解析、展示、过滤需求，具体功能包括：
日志输入：支持两种输入方式
手动粘贴日志文本到文本框；
上传.log/.txt格式的日志文件（分块解析，避免浏览器卡顿）。
日志预处理：可勾选 “去除空行” 选项，自动清理日志中的空行内容。
日志解析与分级展示：
按预设规则（App 启动、进课 / 下课、辅导 / 主讲、辅导看学生等）将日志分级（1-6 级）；
分级日志以树形结构展示，支持展开 / 折叠（标题点击展开 / 折叠层级，正文标题点击展开 / 折叠具体日志）；
提取日志中的时间戳，显示在分级标题旁；
识别视频 / 音频流相关日志，高亮显示关键信息（如 “开启 / 关闭学员 XX 视频 / 音频”）。
日志过滤：内置过滤规则（如隐藏 onLoginResponse 日志），勾选后实时隐藏匹配的日志行。
展示优化：
支持 “自动换行 / 单行展示” 切换，单行展示时日志行可横向滚动；
左右面板可拖动调整宽度（左侧面板宽度范围：150px ~ 容器宽度 70%）；
上传大文件时显示进度条，实时展示解析进度。
辅助操作：
清空输入框 / 清空解析结果按钮；
处理按钮仅在输入有内容时启用（手动粘贴 / 文件上传后自动激活）。
二、UI 界面设计
整体为桌面端适配的单页布局，无响应式（聚焦 PC 端使用），布局结构如下：
整体布局：水平分栏结构，包含左侧组合面板、拖动分隔条、右侧输出面板，高度占满整个视口（100vh），无滚动条溢出。
左侧组合面板（固定初始宽度 420px，可拖动调整）：
上半部分：日志输入面板（固定高度 300px，最小高度 200px）
标题：“日志输入”；
上传区域：“上传日志文件” 按钮 + 隐藏的文件选择框 + “清空输入” 按钮（水平对齐，样式统一）；
进度条区域：默认隐藏，上传文件时显示（进度条 + 进度文本 “解析进度：X%”）；
文本输入框：占满面板剩余高度，占位符 “请粘贴 ohos 格式的日志内容... 或上传日志文件”，等宽字体。
下半部分：功能面板（占左侧面板剩余高度）
“去除空行” 复选框（默认勾选）；
“处理日志” 按钮（默认禁用，有内容时启用，蓝色背景）；
“切换为自动换行” 按钮（绿色背景）；
过滤规则标题：“日志过滤规则”；
过滤规则列表：展示内置过滤项（如 “隐藏 onLoginResponse”），带复选框。
拖动分隔条：宽度 6px，深色背景，显示 “||” 标识，鼠标悬停变为列调整光标。
右侧输出面板（占剩余宽度）：
标题：“分析结果”；
日志输出区域：等宽字体，占满面板剩余高度，展示分级日志树；
“清空结果” 按钮：红色背景，位于输出区域下方。
样式细节：
配色：浅灰背景（#f5f5f5），面板白色背景，按钮区分色（上传 - 紫色、处理 - 蓝色、切换 - 绿色、清空 - 红色）；
分级日志标题样式：1-6 级标题字体大小 / 颜色不同（1 级 18px 红色、2 级 16px 蓝色、3 级 15px 绿色、4 级 13px 橙色、5 级 12px 紫色、6 级 11px 灰色）；
日志行：默认自动换行，切换单行后横向滚动，特殊日志（视频 / 音频）高亮显示描述文本。
三、最终效果
性能层面：
支持处理任意行数的日志（无 5000 行限制）；
上传 GB 级日志文件时，通过 Web Worker 分块解析（256KB / 块），主线程无卡顿，进度条实时更新；
解析后的日志以树形结构展示，展开 / 折叠操作响应即时，无明显延迟。
交互层面：
手动粘贴日志后，“处理日志” 按钮自动启用；
左右面板拖动调整宽度时流畅，无卡顿；
日志分级展开 / 折叠时，DOM 操作批量处理（文档碎片），无闪烁；
过滤规则勾选后，日志行实时隐藏 / 显示，无刷新延迟。
展示层面：
日志分级清晰，不同级别标题样式区分明显；
视频 / 音频流日志关键信息高亮，便于快速识别；
单行 / 自动换行切换生效，适配不同阅读习惯；
大文件上传后，输入框仅显示前 10000 字符（避免卡顿），完整数据缓存到 dataset，处理时使用。
四、使用的技术栈与核心实现
基础技术：
HTML5：语义化标签、File API（文件读取）、Blob（创建 Worker 脚本）、Dataset（缓存完整日志）；
CSS3：Flex 布局（核心布局方式）、CSS 变量（统一缩进 / 字体大小等）、过渡动画（进度条、按钮 hover）、contain 属性（性能优化，限制重绘范围）；
JavaScript（ES6+）：箭头函数、解构赋值、模板字符串、事件监听 / 防抖 / 节流。
核心优化技术：
Web Worker：将大文件解析逻辑放到后台线程，避免阻塞主线程；
分块读取：文件按 256KB 分块读取，降低内存峰值，支持进度反馈；
文档碎片：批量渲染 DOM 节点，减少重绘 / 重排次数；
防抖 / 节流：滚动监听（节流 8ms）、输入框监听（防抖 100ms）、窗口调整（防抖 100ms），降低高频事件触发频率；
正则缓存：提前编译正则表达式，避免重复创建，提升匹配效率；
硬件加速：transform: translateZ (0)，优化面板拖动性能。
核心逻辑：
文件解析：FileReader 分块读取 → Web Worker 处理（清理空行、拆分 OHOS 日志）→ 主线程增量缓存 → 进度更新；
日志分级：正则匹配日志内容 → 构建树形结构 → 递归渲染 DOM；
交互控制：监听输入框内容变化启用按钮、拖动分隔条调整面板宽度、切换换行样式、过滤规则实时生效。
五、关键配置与限制
分块大小：256KB（可配置，注释中标注调整建议：低配设备 16-64KB，高配 64-512KB）；
文件类型：仅支持.log/.txt格式的纯文本文件；
浏览器兼容：适配现代浏览器（Chrome/Firefox/Edge），不支持 IE；
内存优化：大文件解析后，输入框仅显示前 10000 字符，完整数据缓存到 input.dataset.fullData；
布局限制：左侧面板最小宽度 150px，最大宽度为容器宽度的 70%。
关键特征总结（便于后续精准复刻）
核心定位：OHOS 日志专用处理工具，聚焦超大文件解析与分级展示；
核心优化：Web Worker 分块解析（256KB）、无行数限制、按钮自动激活、面板可拖动；
UI 特征：左右分栏 + 拖动分隔条、分级日志树（1-6 级样式区分）、进度条、清空 / 切换按钮；
交互特征：手动粘贴 / 文件上传双输入、去除空行、过滤规则、换行切换、展开 / 折叠。
